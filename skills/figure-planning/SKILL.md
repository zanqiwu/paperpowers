# Figure Planning

Plan figures as communication devices, not decoration.

Read `../../references/figure-patterns.md` before proposing figure types.

## Workflow

1. List the paper's core claims.
2. For each claim, ask whether a figure helps more than text or a table.
3. Choose the smallest set of figures that carries the story.
4. If figure choices are informed by prior papers, verify those papers and read at least their abstracts first.
5. For each chosen figure, produce a brief before discussing rendering.

## Figure Brief Format

For each figure, provide:
- `Goal`
- `Reader question answered`
- `Recommended figure type`
- `Required elements`
- `Optional elements`
- `Common failure modes`
- `Caption intent`

## Common Figure Types

- problem setup
- method overview
- pipeline or data flow
- design space or taxonomy
- qualitative comparison
- ablation summary
- error analysis
- efficiency tradeoff

Use only the types that advance the paper's argument.

## Save Artifact

If useful, save to:
- `docs/paperpowers/figures/YYYY-MM-DD-topic-figure-plan.md`

## Red Flags

- Using both a figure and a table to make the exact same point
- Method diagrams that hide the actual novelty
- Qualitative examples with no stated evaluation purpose
- Decorative icons and arrows that increase confusion
