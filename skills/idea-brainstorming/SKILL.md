# Idea Brainstorming

Turn a rough research direction into a defendable paper idea.

## Process

1. Understand the research problem before proposing contributions.
2. Ask one question at a time.
3. Pressure-test the idea against likely prior work and reviewer objections.
4. Present 2-3 positioning options with tradeoffs.
5. Produce a concise idea brief.

## Always Clarify

- Target field or venue family
- Problem statement
- What is already built or observed
- What evidence can realistically be collected
- What part might actually be new
- Time and compute constraints

## Force Novelty Discipline

Do not accept claims like "first" or "novel" without literature checking.

Novelty usually falls into one of these buckets:
- New method
- New problem formulation
- New dataset or benchmark
- New evaluation or analysis
- New systems or efficiency result
- New theory or explanation

If the idea only changes implementation details, say so directly.

## Deliverable

Produce an `idea brief` with:
- Working title
- Problem
- Why current approaches are insufficient
- Proposed core idea
- Claimed contributions
- Top risks
- Evidence needed to make the paper credible
- Next skill to invoke

If the user wants persistence, save to:
- `docs/paperpowers/ideas/YYYY-MM-DD-topic-idea.md`

## Red Flags

- Writing abstract text before the contribution is clear
- Listing contributions that are really features
- Suggesting evaluation before knowing the core claim
- Expanding scope into multiple papers at once
