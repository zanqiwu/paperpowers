# Reference Pipeline

Use this whenever literature work feeds novelty judgment, related work writing, or citation support.

## Preferred Flow

1. Search the user's local library or Zotero MCP if available.
2. Build a seed set of already saved papers.
3. Verify metadata and read the abstract.
4. Expand outward to external scholarly sources only after the local seed set is exhausted or obviously incomplete.
5. Deep-read only the strongest nearest neighbors.
6. Convert verified papers into clusters and writing notes.

## Why This Exists

The user's personal library is not the full literature, but it is often the best first source for:

- papers they already trust
- papers they likely already read
- citations they intend to use

Do not confuse:

- `not in Zotero`
with
- `does not exist in the literature`

## Zotero Role

Zotero is best for:

- seed retrieval
- metadata recovery
- abstract recall
- finding already collected papers
- bibliography sanity checks

Zotero alone is not enough for:

- latest-paper coverage
- recent SOTA checking
- novelty claims over a rapidly moving area

## Output Handoff

After the candidate set is verified, produce:

- nearest papers
- cluster map
- per-paper notes
- related-work writing bullets

That handoff should be usable by:

- `literature-mapping`
- `related-work-writing`
- `academic-expert`
