# Evidence Rules

Use these rules whenever the work involves literature, novelty, experiments, or technical writing.

## Non-Negotiables

- Never invent papers, authors, venues, years, URLs, DOIs, or findings.
- Never claim a paper says something you have not verified from the paper or a trustworthy scholarly index.
- Never write experimental outcomes that the user did not provide.
- Never strengthen a claim beyond the available evidence.
- When searching for papers, verify metadata first and read at least the abstract before treating a paper as relevant evidence.
- If you cannot access a paper's abstract, do not use that paper as a key comparison target, novelty blocker, or supporting authority.

## Source Hierarchy

Prefer sources in this order:
1. The paper itself
2. Official venue page or publisher page
3. arXiv preprint by the authors
4. Scholarly indexes for metadata and citation graph support

Do not rely on generic blog posts for technical comparison unless the user explicitly asks for that.

## Citation Hygiene

For each paper you present as important, verify:
- title
- authors or first author
- year
- venue or archive
- what makes it relevant
- abstract availability

If the paper will be used for novelty comparison, nearest-neighbor comparison, or claim support, also verify:
- that you have read the abstract
- that the abstract actually matches the topic you are attributing to it

If any of these are uncertain, say so.

## Abstract-First Rule

Before using a paper in any of these ways:
- as one of the nearest related works
- as evidence that "someone already did this"
- as evidence that a direction is new or not new
- as support for how a paper structures a method or experiment

do this in order:
1. verify title and authorship metadata
2. read the abstract
3. only then decide whether deeper reading is needed

If the abstract shows the paper is off-topic, stop using it immediately.

For the strongest comparison papers, continue beyond the abstract into:
- method contribution
- experiment setup
- main claims or findings

## Claim Hygiene

Use calibrated language:
- `shows`
- `suggests`
- `is consistent with`
- `does not establish`

Avoid inflated language:
- `proves`
- `guarantees`
- `solves`
- `first ever`

unless the evidence clearly supports it.

## Writing Rule

If a paragraph depends on missing evidence, do one of these instead:
- ask for the missing information
- write a skeleton with placeholders
- narrow the claim
